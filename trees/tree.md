# Trees (arvores)

## Explicação
- Arvores são estruturas de dados abstratas e não lineares que consistem em uma serie de nós (nodes) que são conectados entre sí para formar uma representação hierarquica de dados.
- É semelhante a uma lista ligada mas um node pode apontar para multiplos nodes e não somente para um como nas listas ligadas simples.
- Cada nó em uma arvore pode ser também chamado de vertice e as conexões entre os vertices são chamados de arestas (edges).

### Algumas nomenclaturas das estruturas de uma arvore.
- <b>Root node</b> - é o node que fica no topo de uma arvore
- <b>Child node</b> - é um node que possui uma aresta que o liga a outro node que está um nivel acima dele
- <b>Parent Node</b> - é um node que contem um ou mais child nodes conectados a ele
- <b>Leaf node</b> - é um node que não possui nenhum child node
- <b>Height</b> - representa a altura de uma arvore que conta o maior caminho possivel do root node ao um leaf node
- <b>Depth</b> - a profundidade é uma caracteristica dos nodes e que conta o numero de arestas (edges) de um determinado node até o root node.
